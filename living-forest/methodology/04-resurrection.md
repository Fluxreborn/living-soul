# 04 - 复活机制 / Resurrection Mechanism

## 核心原则

**废弃 ≠ 删除**

Graveyard 是「冷冻舱」，不是「焚化炉」。

## 废弃流程

当一个分支需要废弃时：

### 1. 评估
```markdown
分支：005-回声网络
评估日期：2026-03-23

废弃原因：
- 技术复杂度超出当前能力
- 需要向量数据库支持，但尚未就绪
- 实现周期 > 2 周，与快速迭代目标冲突

复活条件：
- 向量数据库技术成熟
- 或：项目周期允许长期投入

建议：废弃，进 Graveyard，等待条件成熟
```

### 2. 迁移
```bash
# 1. 移动文件
mv trunk/005-回声网络.md graveyard/005-old-回声网络.md

# 2. 更新 _tree.md
# - 从主干移除 005
# - 添加到 Graveyard 表格
# - 标记复活条件
```

### 3. 记录
在 `_tree.md` 中：
```markdown
## 墓地（Graveyard）
| 编号 | 名称 | 废弃日期 | 原因 | 复活条件 |
|------|------|---------|------|---------|
| 005-old | 回声网络 | 2026-03-23 | 技术复杂度过高 | 向量数据库就绪 |

## 演化历史
- **v2** (2026-03-23)：废弃 005-old
  - 原因：技术复杂度过高，影响迭代速度
  - 决策人：Fable
  - 复活条件：向量数据库技术就绪
```

## 复活流程

当复活条件满足时：

### 1. 识别
```markdown
日期：2026-03-25
事件：向量数据库技术成熟
触发：检查 Graveyard 中标记「向量数据库就绪」的项目
发现：005-old 回声网络 符合复活条件
```

### 2. 决策
```markdown
复活评估：
- 原方案（回声网络）：复杂的网络效应模拟
- 新需求：向量数据库迁移
- 判断：不是简单复活，而是重生为不同形态
- 决策：给予新编号 006，继承灵魂但改变肉身
```

### 3. 执行
```bash
# 1. 创建新分支（新编号）
cp graveyard/005-old-回声网络.md branches/006-向量数据库迁移.md

# 2. 更新内容（根据新需求调整）
# 编辑 006-向量数据库迁移.md

# 3. 更新 _tree.md
# - 在 Branches 添加 006
# - 在 Graveyard 更新 005-old 状态：已复活为 006
# - 在复活记录添加条目
```

### 4. 记录
```markdown
## 分支（Branches）
- `006-向量数据库迁移` ← 从 005-old 复活重生
  - 状态：研究中
  - 分叉日期：2026-03-25
  - 决策人：Fable + Karo
  - 与原方案差异：简化网络效应，专注向量检索

## 墓地（Graveyard）
| 编号 | 名称 | 废弃日期 | 原因 | 状态 |
|------|------|---------|------|------|
| 005-old | 回声网络 | 2026-03-23 | 技术复杂度过高 | 🌱 已复活为 006 |

## 复活记录
- **2026-03-25**：006-向量数据库迁移 → 从 005-old 复活
  - 复活条件满足：向量数据库技术就绪
  - 重生差异：简化网络效应，专注向量检索
  - 保留价值：原方案的核心洞察（语义相似度的重要性）
```

## 复活 vs 新建

什么时候该复活，什么时候该新建？

| 情况 | 操作 | 示例 |
|------|------|------|
| 复活条件满足，方案基本可用 | **复活** | 技术成熟后用旧方案 |
| 复活条件满足，但需求已变 | **重生**（新编号） | 向量数据库迁移 |
| 类似想法，但完全重新设计 | **新建** | Token Saver → Soul Memory（独立项目） |

## Graveyard 的维护

定期（每周/每月）检查：

```markdown
## Graveyard 审查记录
日期：2026-03-30

| 编号 | 名称 | 废弃日期 | 复活条件 | 条件状态 | 行动 |
|------|------|---------|---------|---------|------|
| 005-old | 回声网络 | 2026-03-23 | 向量数据库就绪 | ✅ 已满足 | 已复活为 006 |
| 006-old | 情绪图谱 | 2026-03-23 | 与亮度系统合并 | ⏸️ 永久废弃 | 归档，不再审查 |
| 007-old | 某某功能 | 2026-03-20 | 等待用户反馈 | ⏳ 未满足 | 继续等待 |
```

## 与 Soul Memory 的结合

Graveyard 中的分支也是 Soul Memory 的「签」：

```json
{
  "id": "005-old",
  "content": "回声网络：用网络效应模拟记忆关联",
  "dimensions": {
    "visual": "复杂的节点连线图",
    "emotion": "期待→挫折",
    "brightness": 0.3,
    "body": "颈椎僵硬，盯着复杂代码"
  },
  "status": "graveyard",
  "resurrection_condition": "向量数据库就绪"
}
```

当条件满足时，这支签的亮度会自动提升，提示「该复活了」。
