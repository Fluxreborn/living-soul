# 07 - 记忆分层与压缩策略 / Memory Layering & Compression

## 双重记忆系统定位

```
┌─────────────────────────────────────────────────────────────┐
│                    记忆总览                                  │
├─────────────────────────┬───────────────────────────────────┤
│      Soul Memory        │        Living Forest              │

├─────────────────────────┼───────────────────────────────────┤

## 核心原则：预加载文件大小控制

### 问题
如果所有项目（无论活跃还是归档）都详细加载，预加载文件会**无限膨胀**。

### 解决方案：三级压缩

```
项目生命周期 → 预加载内容变化

立项时    [详细]   → 完整 _tree.md（活跃项目）
    ↓
进行中    [详细]   → 完整 _tree.md（活跃项目）
    ↓
归档时    [压缩]   → 精简条目（一句话）
    ↓
历史项目  [删除]   → 从预加载中移除，保留在 _forest.md 索引
```

## 压缩规则详解

### Level 1：活跃项目（完整）

**触发条件**：进行中、待命状态

**预加载内容**：
- `_tree.md` 完整内容（~500-1000 tokens）
- `_lineage.md`（如有跨项目依赖）

**示例**：
```markdown
## Soul Memory [🔄]
```tree
Soul Memory
├── 001 立项 [✅]
├── 002 抽签机 [🔄]
│   ├── 002.1 签数生成 [✅]
│   └── 002.2 梦境输出 [🔄]
└── ...
```
当前阻塞：无
关键产出：soulos/soul_memory_simple.py
```

### Level 2：归档项目（一句话）

**触发条件**：已完成、已归档状态

**预加载内容**：
- 仅一句话（~50 tokens）
- 保留项目文件夹，但不加载详情

**示例**：
```markdown
- Token Saver — 已归档，分娩出 Soul Memory，防御补丁→根本探索
- Trade Services 山西 — 已完成，山西贸易服务研究报告，2026-02 交付
```

### Level 3：历史项目（仅目录）

**触发条件**：归档超过 3 个月，或项目明确结束

**预加载内容**：
- 仅从 `_forest.md` 中移除，保留在 `archive/` 文件夹
- 如需查阅，手动读取

**迁移流程**：
```bash
# 项目归档 3 个月后
mv projects/trade-services/ projects/archive/2026-02-trade-services/

# 更新 _forest.md
# 从"归档项目"表格移除
# 添加到"历史项目"目录列表
```

## 预加载文件大小控制目标

| 文件 | 当前大小（估算） | 目标大小 | 控制手段 |
|------|----------------|---------|---------|
| ACTIVE.md | ~500 tokens | < 1000 tokens | 只保留活跃项目 |
| _forest.md | ~1000 tokens | < 1500 tokens | 归档项目一句话 |
| Soul Memory | ~800 tokens | ~800 tokens | 活跃项目完整保留 |
| Living Forest | ~600 tokens | ~600 tokens | 活跃项目完整保留 |
| **总计** | **~3000 tokens** | **< 4000 tokens** | **动态压缩** |

## 实施检查清单

### 立即执行
- [ ] 创建新的 `ACTIVE.md`（替代两天的 memory 文件）
- [ ] 精简 `_forest.md`，归档项目只保留一句话
- [ ] 将旧项目移至 `projects/archive/`

### 日常维护
- [ ] 项目状态变更时，更新 `ACTIVE.md`
- [ ] 项目归档时，执行压缩（完整→一句话）
- [ ] 项目历史化时，移至 archive/

### 长期规则
- [ ] 活跃项目 > 10 个时，评估是否过多
- [ ] 归档项目 > 20 个时，批量历史化
- [ ] 每季度清理一次 `ACTIVE.md`

## 与 Soul Memory 的配合

```
用户对话
    ↓
加载 SOUL.md + USER.md（人格）
    ↓
加载 ACTIVE.md（短期工作焦点，已压缩）
    ↓
加载 _forest.md（项目索引，已压缩）
    ↓
用户提到具体项目
    ↓
按需加载 projects/<name>/_tree.md（活跃）
或
从 Soul Memory 抽签浮现（日常经验）
```

**结果**：预加载始终保持精简，详细信息按需获取。
