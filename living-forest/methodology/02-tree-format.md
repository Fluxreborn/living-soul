# 02 - 树形格式规范 / Tree Format Specification

## _tree.md 标准结构

```markdown
# <项目名称>

> 生成时间：YYYY-MM-DD HH:mm
> 版本：v{N}

## 项目根
**<一句话描述>**
目标：<49字内的核心目标>

## 主干（Trunk）
```tree
<项目名称>
├── 001 <分支名> [<状态>]
│   └── <简要描述>
├── 002 <分支名> [<状态>]
│   ├── 002.1 <子任务> [<状态>]
│   └── 002.2 <子任务> [<状态>]
└── 003 <分支名> [<状态>]
```

## 分支（Branches）
- `<编号>-<名称>` ← 从主干 <X> 延伸
  - 状态：<研究中/原型中/待验证>
  - 分叉日期：YYYY-MM-DD
  - 决策人：<名字>

## 墓地（Graveyard）
| 编号 | 名称 | 废弃日期 | 原因 | 复活条件 |
|------|------|---------|------|---------|
| <编号>-old | <名称> | YYYY-MM-DD | <原因> | <条件> |

## 演化历史
- **v1** (YYYY-MM-DD)：初始结构（<N> 个分叉）→ 见 `_tree.v1.md`
- **v2** (YYYY-MM-DD)：<变化描述> → 见 `_tree.v2.md`
- **v{N}** (YYYY-MM-DD)：<当前变化>（当前）

## 复活记录
- **YYYY-MM-DD**：<新编号>-<新名称> → 从 <旧编号>-old 复活
  - 复活原因：<描述>
```

## 状态标记规范

| 标记 | 含义 | 视觉 |
|------|------|------|
| ✅ / [完成] | 已完成 | 绿色 |
| 🔄 / [进行中] | 正在执行 | 黄色 |
| ⏸️ / [待命] | 暂停，等待条件 | 灰色 |
| 🧪 / [研究中] | 探索阶段 | 蓝色 |
| ⚠️ / [阻塞] | 有阻塞项 | 红色 |
| 🪦 / [废弃] | 在 Graveyard 中 | 灰色删除线 |

## 树形缩进规则

```
主干节点（001, 002...）
├── 直接子任务（002.1, 002.2...）
│   └── 孙任务（002.1.1...）—— 尽量少用三层以上
└── 另一个子任务

废弃标记（在 Graveyard 表格中）
编号-old：表示这是废弃版本，新编号可能已复活
```

## 演化历史写法

每次结构变化时：
1. 备份旧树：`cp _tree.md _tree.v{N}.md`
2. 在 `_tree.md` 顶部更新版本号
3. 在「演化历史」添加新条目
4. 描述变化的**原因**，不只是**结果**

好的演化记录：
```markdown
- **v2** (2026-03-24)：4 主干 + 1 分支 + 2 废弃
  - 废弃 005-old「回声网络」：技术复杂度过高
  - 废弃 006-old「情绪图谱」：与 003 亮度系统重叠，合并
  - 新增分支 006「向量数据库迁移」：从 005-old 复活
```

差的演化记录：
```markdown
- **v2** (2026-03-24)：修改了结构
```

## 文件命名规范

| 文件 | 命名 | 说明 |
|------|------|------|
| 活树主干 | `_tree.md` | 下划线开头，表示元数据 |
| 历史快照 | `_tree.v{N}.md` | v1, v2, v3... |
| 血统声明 | `_lineage.md` | 记录从哪来 |
| 具体分支 | `{编号}-{名称}.md` | 如 `002-抽签机.md` |
| 废弃分支 | `graveyard/{编号}-old-{名称}.md` | 保留完整内容 |
