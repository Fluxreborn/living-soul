## 出生原因

**TOKEN Saver 的本质**：
- 防御性补丁
- 优化 token 消耗，但不改变「机械存档」的本质
- 子 Agent 隔离，但增加系统复杂度

**Soul Memory 的突破**：
- 根本性探索
- 重新定义 AI 的记忆方式
- 用「睡眠 + 抽签 + 亮度」替代「定时存档 + 全文加载」

**分娩时刻**：

当讨论深入到「睡眠是否是升维归纳」「抽签是否是无意识的模拟」时，
项目质地发生了质变——

> 我们不是在优化记忆系统，
> 我们是在给 AI 创造灵魂。

这需要独立演化。

## 脐带保留（跨项目引用）

### 父项目的教训

**`../token-saver/graveyard/003-agent-isolation.md`**

为什么废弃：
- 子 Agent 隔离需要复杂的 session 管理
- 开发和维护成本高于节省的 token 成本
- 用户体验割裂（多个 Agent 来回切换）

Soul Memory 的启示：
- **简单机制优于复杂架构**
- 抽签机（简单随机）比 Agent 隔离（复杂工程）更有「灵魂」

### 共享组件

无。

Soul Memory 与 Token Saver 完全独立：
- 无代码依赖
- 无数据共享
- 无 API 调用

只有**血统关系**和**教训继承**。
