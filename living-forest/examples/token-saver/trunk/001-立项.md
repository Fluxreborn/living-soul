# 001-立项.md

## 项目背景

OpenClaw 的 token 消耗过高：
- 每次 session 加载大量上下文
- 没有精确的成本归因
- 子 Agent 机制增加了额外开销

## 目标

1. **计量**：精确统计每个 session 的 token 消耗
2. **归因**：识别高成本环节
3. **优化**：通过子 Agent 隔离降低成本

## 方案概述

### KPI 计量（方案 A）
- 追踪每个 session 的 input/output tokens
- 生成可视化报表
- 识别异常消费模式

### 子 Agent 隔离（方案 B）
- 简单任务分配给轻量级 Agent
- 复杂任务分配给主力 Agent
- 通过 session 路由自动分配
