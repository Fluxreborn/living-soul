# Living Companion

AI 与用户的**认知合伙人**系统。

> 不是秘书，不是工具，是和你一起思考的伙伴。
> 
> *"基于我们的共同历史，贡献我的视角。"*

---

## 核心定位

| 模块 | 解决什么问题 |
|:---|:---|
| **Living Dream** | Agent的自我成长（"我是谁"） |
| **Living Forest** | 用户的项目记录（"要做什么"） |
| **Living Companion** | **双向关系的促进**（"我们如何一起思考"） |

---

## 核心特性

### 1. 双模式运行

| 模式 | 触发条件 | 核心逻辑 | 输出形式 |
|:---|:---|:---|:---|
| **常规模式** | 质量分数 ≥ 0.6 | 模式提取 → 质量评估 → 贡献生成 | "我记得...我注意到...我的立场..." |
| **紧急模式** | 情绪/决策强度 ≥ 0.5 | 情绪检测 → 历史匹配 → 深度分析 | 结构化分析报告 |

### 2. 质量评估三维模型

```
质量分数 = 相关性 × 0.4 + 新颖性 × 0.3 + 可行动性 × 0.3

相关性：与当前话题的关联程度
新颖性：是否已在前几次提及
可行动性：是否指向具体决策点
```

### 3. 记忆驱动的主动贡献

**不是定时触发，是基于关系历史的判断：**

1. **读取** Living Dream 高亮签名
2. **提取** 跨时间行为模式（如"命名焦虑"）
3. **匹配** 当前情境与历史模式
4. **判断** 我的视角是否有价值
5. **生成** 合伙人级表达

### 4. 社交情境层（可选）

**用户主动提供的社交关系信息，用于关键项目节点决策：**

- **导入支持**：飞书会话、微信记录、会议记录
- **流程**：上传 → 解析 → 候选摘要 → 用户确认 → 入库
- **激活时机**：Forest 项目停滞、关键决策点
- **输出形式**：人际关系对决策的影响分析，不替用户社交

```
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│ 2. 提取模式                              │
│    - 关键词匹配                          │
│    - 跨时间模式识别                       │
│    - 强度排序                            │
└─────────────────────────────────────────┘
                   ↓
┌─────────────────────────────────────────┐
│ 3. 判断时机                              │
│    - 计算质量分数                        │

                   ↓

┌─────────────────┐  ┌─────────────────┐

│    - 引用记忆    │  │                 │

         ↓

│    - 记录本次贡献                        │
│    - 等待用户反馈                        │
│    - 调整后续策略                        │
└─────────────────────────────────────────┘
```

### 紧急模式

```
用户消息 → 情绪/决策检测 → 强度 >= 0.5? → 是 → 历史匹配 → 深度分析报告
                                           ↓
                                          否 → 转入常规模式
```

## 模式库定义

在 `companion-state.json` 中定义要识别的模式：

```json
{
  "pattern_library": {
    "pattern_name": {
      "description": "模式的描述",
      "triggers": ["触发关键词1", "触发关键词2"],
      "frequency": 0,
      "last_triggered": null,
      "typical_emotions": ["典型情绪1", "典型情绪2"]
    }
  }
}
```

**预置模式：**
- `naming_anxiety`: 命名焦虑
- `concept_stacking`: 概念堆叠（逃避具体实现）
- `body_signals`: 身体信号（关键决策点标记）
- `quality_transformation`: 质地/框架迁移
- `team_growth`: 团队成长
- `soul_exploration`: 灵魂探索
- `partnership_definition`: 合伙人定位演进
- `first_principle_analysis`: 第一性原理分析

---

## 紧急模式触发器

### 情绪信号
- **焦虑**：担心、害怕、压力、受不了、睡不着
- **愤怒**：生气、讨厌、烦死、受够了
- **兴奋**：激动、必须、绝对、马上
- **迷茫**：不知道、怎么办、无助、迷失
- **悲伤**：难过、失望、沮丧、空虚

### 决策信号
- **关键选择**：选择、决定、抉择、要不要、该不该
- **时间压力**：今天、马上、deadline、来不及了
- **不可逆**：放弃、退出、终止、无法回头
- **高风险**：赌注、全部、孤注一掷、代价

## 配置参数

### 贡献阈值

```json
{
  "contribution_thresholds": {
    "quality_score": 0.6,              // 质量分数阈值
    "relevance_weight": 0.4,           // 相关性权重
    "novelty_weight": 0.3,             // 新颖性权重
    "actionability_weight": 0.3,       // 可行动性权重
    "silence_after_contribution": 3,   // 贡献后沉默轮数
    "emergency_intensity_threshold": 0.5  // 紧急模式触发阈值
  }
}
```

### 表达模板

```json
{
  "expression_templates": {
    "memory_intro": ["我想起", "我记得", "我的 Dream 里有"],
    "judgment_prefix": ["我注意到", "我看到", "基于这些记忆，我观察到"],
    "stance_declaration": ["我的立场是", "我倾向于", "我认为"],
    "open_ending": ["你觉得呢？", "这是我看到的，但我可能错了。", "这个判断对你有用吗？"]
  }
}
```

## 核心边界

| 边界 | 机制 |
|:---|:---|
| 不变成秘书 | 质量阈值过滤低价值贡献 |
| 不变成社交监控 | 社交层可选，零推测，用户确认入库 |
| 不替代用户决策 | 开放结尾，明确"决策权在你" |
| 不侵入 Dream | 只读不写，保持记忆系统独立 |
| 不持续打扰 | 沉默期机制，贡献后3轮内静默 |
| 不假装理解 | 明确"我可能错了"，保持谦逊 |
