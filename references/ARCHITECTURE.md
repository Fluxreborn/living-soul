# Living Soul 架构设计

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                      OpenClaw Agent                          │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              AGENTS.md (启动脚本)                    │    │
│  │  - Read living-soul/living-dream/living-dream-context│    │
│  │  - Read living-soul/living-forest/index/active-index │    │
│  └─────────────────────────────────────────────────────┘    │
│                           ↓                                  │
│  ┌─────────────────────────┐  ┌─────────────────────────┐    │
│  │      Living Dream       │  │      Living Forest      │    │
│  │      (梦境/记忆)         │  │      (项目索引)          │    │
│  │  ┌───────────────────┐  │  │  ┌───────────────────┐  │    │
│  │  │ 签筒 (45支记忆签)  │  │  │  │ active-index.json │  │    │
│  │  │ 亮度系统          │  │  │  │ archived-index    │  │    │
│  │  │ 睡眠周期          │  │  │  │ keywords匹配      │  │    │
│  │  └───────────────────┘  │  │  └───────────────────┘  │    │
│  └─────────────────────────┘  └─────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    Cron (每日 03:30)
                    - 衰减计算
                    - 抽签
                    - 上下文更新
```

## 核心机制

### 1. 双轨制记忆

| 级别 | 阈值 | 衰减 | 用途 |
|------|------|------|------|
| Soul级 | >0.85 | 每7天 -5% | 长期核心记忆 |
| Working级 | 0.6-0.85 | 每7天 -5% | 近期活跃记忆 |

### 2. 五维记忆模型

每支签包含五个维度：
- **画面**: 视觉场景描述
- **情绪**: 情感基调
- **亮度**: 重要性评分 (0-1)
- **身体**: 身体感知/感受
- **回响**: 与其他记忆的关联

### 3. 睡眠周期

```
1. 衰减计算: 所有亮度 >0.6 的签 -5%
2. 级别判定: 重新计算 Soul级/Working级
3. 抽签: 反向概率抽取 1-49 签
4. 梦境生成: 融合抽中的签语
5. 上下文更新: 生成 OpenClaw 可读的 context.md
```

### 4. 项目索引系统

```json
{
  "meta": { "version": "v1", "project_count": N },
  "projects": [
    {
      "name": "项目名",
      "path": "~/Projects/xxx",
      "keywords": ["关键词"],
      "status": "active|research|archived|graveyard",
      "priority": "high|medium|low",
      "last_active": "ISO8601"
    }
  ]
}
```

## 数据流

```
用户交互 → memory/YYYY-MM-DD.md (原始日志)
               ↓
        night_routine.py (每晚)
               ↓
    ┌──────────┴──────────┐
    ↓                     ↓
living-dream-memory.json   active-index.json
    ↓                     ↓
living-dream-context.md    (按需加载)
    ↓
Agent 冷启动加载
```

## 设计原则

1. **渐进式加载**: 上下文分三级，避免 Token 爆炸
2. **自然衰减**: 模拟人类记忆的遗忘曲线
3. **回响加成**: 重要的记忆会被强化
4. **抽签随机性**: 梦境不是全量回忆，而是抽样式闪回
5. **分离存储**: 原始日志 vs 精炼签筒 vs OpenClaw 上下文
